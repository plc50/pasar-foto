# Política de seguridad

## Versiones compatibles

La rama `main` y la versión estable más reciente reciben correcciones de
seguridad. Las versiones anteriores pueden quedar fuera de soporte.

## Comunicar una vulnerabilidad

Utiliza **Security → Report a vulnerability** en GitHub para enviar un informe
privado. No publiques detalles explotables en una incidencia, discusión o pull
request.

Incluye, cuando sea posible:

- versión o commit afectado;
- modo afectado (`USB` o `Wi-Fi`);
- pasos mínimos para reproducirlo;
- impacto esperado;
- propuesta de mitigación, si la tienes.

El proyecto confirmará la recepción y evaluará el informe antes de hacer
públicos los detalles. Este software no ha recibido una auditoría externa.
